Socket Class Diagram (UML rules)
================================

Purpose
-------
Formal class-diagram representation following UML class-diagram rules (see
https://en.wikipedia.org/wiki/Class_diagram). Focus: classes, attributes
with visibility and types, operations with parameters and return types,
generalization (inheritance), interface realization, associations with
multiplicity, aggregation and composition, and dependencies.

Notation used
-------------
- Visibility: `+` (public), `-` (private), `#` (protected)
- Multiplicity: `1`, `0..1`, `*`, `1..*`, `0..*`
- Associations shown with role names and multiplicities when useful

Key relationships (UML semantics)
--------------------------------
- `dispatch::IDeviceHandler` <<interface>> <-|.. `dispatch::WmosHandler` : <<realize>>
- `dispatch::IDeviceHandler` <<interface>> <-|.. `dispatch::UnknownHandler` : <<realize>>
- `dispatch::DeviceDispatcher` o-- `dispatch::IDeviceHandler` : "0..* handlers" (aggregation)
- `service::PiBService` *-- `transport::HttpServer` : "1 wmosServer" (composition)
- `service::PiBService` o-- `transport::SocketConnection` : "0..1 upstream" (aggregation)
- `service::PiBService` *-- `dispatch::DeviceDispatcher` : "1 dispatcher" (composition)
- `service::PiAService` *-- `transport::SocketConnection` : "1 peer" (composition)
- `transport::HttpServer` 1 -- * `transport::ThreadPoolWorker` : "creates/assigns" (association)
- `transport::SocketConnection` --> `protocol::JsonMessage` : "exchanges (dependency)"
- `transport::HttpServer` --> `protocol::JsonMessage` : "parses (dependency)"

Notes on UML correctness
------------------------
- Attributes and operations include visibility and types (UML rule).
- Interfaces are shown as `<<interface>>` and realized by concrete handlers.
- Composition (`*--`) denotes strong ownership (life-cycle bound).
- Aggregation (`o--`) denotes shared or optional relationships.
- Associations between server and worker show multiplicity: one server can have many workers.
- Dependencies use `-->` to show transient usage (parsing/exchanging messages).
